Continuum is a browser extension and related backend service that helps users capture browser-based workflows, create process documentation, collect evidence such as screenshots, and optionally generate meeting transcripts, notes, and guided documentation.

This Privacy Policy explains what information Continuum collects, how it is used, when it is shared, and what choices users have.

---

## 1. What Continuum Does

Continuum is designed to help users document workflows they perform in the browser. Depending on which features a user chooses to use, Continuum may:

- Observe user-initiated workflow actions on webpages
- Save process metadata and generated documentation
- Capture screenshots when the user requests them
- Record meeting audio when the user starts meeting capture
- Generate transcripts, summaries, and notes
- Generate draft guides using trusted public documentation sources

---

## 2. Information We Collect

Continuum collects only the information needed to provide its user-facing features.

### A. Settings and Local Extension Data

Continuum stores certain settings and temporary state in Chrome extension storage, such as:

- Backend URL
- Documentation preferences
- Audit or document-generation rules
- Meeting notes style
- Temporary session, process, and capture state

### B. Workflow Capture Data

When a user starts using Continuum to document a workflow, Continuum may collect:

- Page URL
- Page title
- Click, change, and submit events
- Element labels and simplified selectors
- Field names and input types
- Limited value previews for certain changed form fields

In the current build, Continuum is intended not to capture full values for sensitive field types such as passwords, hidden fields, phone fields, number fields, and email fields in standard change previews. Users should still avoid using Continuum on pages containing highly sensitive personal, financial, health, or confidential information unless they are comfortable with that data being processed as part of the workflow they are documenting.

### C. Screenshots

If a user chooses to capture screenshots, Continuum collects and stores image data of the visible browser tab together with related metadata such as page URL, page title, caption, session ID, and step index.

### D. Meeting Capture Data

If a user starts meeting capture, Continuum may collect and store:

- Tab audio
- Microphone audio, if available and permitted
- Page URL and page title
- Uploaded meeting audio files
- Transcript text
- AI-generated notes, minutes, summaries, action items, and follow-up questions

Meeting capture is optional and only occurs when the user actively starts it.

### E. Generated Content

Continuum may store generated outputs and related evidence, including:

- Process documents
- Quick reference guides
- Control narratives
- Meeting notes and minutes
- Guided run content
- Evidence summaries

### F. Trusted External Source Feature Inputs

If a user uses the trusted external documentation feature, Continuum may collect and process:

- Topic
- Audience
- Notes
- Questions
- Source URLs entered by the user
- Content retrieved from approved trusted public documentation sources

## 3. How We Use Information

We use the collected information to:

- Provide workflow capture and documentation features
- Generate process drafts, notes, and guided content
- Store evidence linked to a workflow session
- Create optional transcripts and meeting notes
- Answer questions about generated or trusted external documentation
- Save user settings and maintain product functionality
- Troubleshoot, secure, and improve the reliability of the service

We do not use collected user data for personalized advertising.

---

## 4. How Information Is Shared

We do not sell user data.

We may share user data only as needed to operate Continuum and provide its features, including with the following categories of recipients:

### A. Our Backend and Hosting Providers

Data sent by the extension is processed by the Continuum backend and may be stored by our cloud hosting or infrastructure providers, such as [Hosting Provider Name, e.g. Render].

### B. AI Service Providers

If AI-powered features are enabled on the backend, Continuum may send relevant content to OpenAI to provide:

- Audio transcription
- Meeting note generation
- Document generation or question-answering features, where configured

This may include uploaded meeting audio, transcript text, user prompts, source text, and generated-document context.

### C. Search and Trusted Documentation Providers

If the trusted external documentation feature is used, Continuum may send user-entered topics, notes, or related search terms to search providers or request approved documentation pages from trusted public sources in order to retrieve relevant source material.

Depending on the feature flow, this can include services such as:

- Bing
- DuckDuckGo
- learn.microsoft.com
- support.microsoft.com
- support.apple.com
- support.google.com
- help.openai.com
- platform.openai.com
- developers.openai.com

### D. Legal, Security, and Abuse-Prevention Disclosures

We may disclose information if required by law, regulation, legal process, or enforceable governmental request, or when reasonably necessary to detect, prevent, or address fraud, abuse, security, or technical issues.

## 5. Data Retention

Continuum may retain stored sessions, screenshots, uploaded meeting files, transcripts, notes, and generated documents until they are deleted by us, deleted through future product controls, or removed as part of infrastructure cleanup or service changes.

The current beta build may not yet include full self-service deletion controls inside the product. Users who want their stored backend data removed can contact us at [Support Email].

Chrome extension settings and temporary state stored locally remain in the user’s browser until removed by the user, overwritten, or cleared by uninstalling the extension or clearing extension data.

## 6. User Choices and Controls

Users can choose whether to use optional features such as screenshots, meeting capture, and trusted external documentation features.

Users may also:

- Uninstall the extension at any time
- Stop using the service at any time
- Avoid starting optional capture features
- Contact us to request deletion of backend-stored data, where applicable
